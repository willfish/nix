# Daily Notes

Use this for maintaining `~/Notes/YYYY-MM-DD/today.md`.

**Read first:**
- `references/daily-notes.md` — Location, template format, the exact workflow (read yesterday's note first, summarise completed work concisely, leave "plan for today" blank unless told otherwise), and style rules (use PR numbers not full URLs, group related work, keep entries to one line).

Workflow:
1. Read yesterday's note to see what was planned and what PRs were open.
2. Summarise completed work under "What I did yesterday".
3. Leave the "What I plan to do today" section blank unless the user has given specific tasks.
4. Keep everything concise — one line per item.
